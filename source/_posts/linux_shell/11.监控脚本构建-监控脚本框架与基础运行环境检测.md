## 监控脚本框架与基础运行环境检测

**本节课目标：** 搭建脚本基础框架，复习变量、函数及 `if` 判断，并集成 `vmstat` 工具。

### 工具介绍：vmstat

`vmstat` 是监测 CPU、内存、I/O 等系统性能的基础工具。

- **常用命令**：`vmstat 5 1` (每 5 秒输出一次，仅 1 次)。

#### vmstat 输出字段详解

执行 `vmstat` 命令后，输出结果各列含义如下：

- **Procs（进程）**
    - `r` (runnable): 等待运行的进程数。如果长期大于 CPU 核心数，说明 CPU 资源饱和。
    - `b` (blocked): 处在不可中断睡眠状态的进程数，通常是在等待 I/O。
- **Memory（内存）**
    - `swpd`: 使用的虚拟内存（Swap）总量（KB）。如果该值不断变化，说明内存不足。
    - `free`: 空闲的物理内存总量（KB）。
    - `buff`: 用作缓冲区的内存总量（KB），用于合并磁盘写入。
    - `cache`: 用作缓存的内存总量（KB），用于加速文件读取。
- **Swap（交换分区速度）**
    - `si` (swap-in): 每秒从磁盘交换到内存的数据量（KB/s）。
    - `so` (swap-out): 每秒从内存交换到磁盘的数据量（KB/s）。
- **IO（磁盘读写速度）**
    - `bi` (block-in): 每秒从块设备接收到的块数（读操作）。
    - `bo` (block-out): 每秒发送到块设备的块数（写操作）。
- **System（系统）**
    - `in` (interrupt): 每秒中断次数，包括时钟中断。
    - `cs` (context switch): 每秒上下文切换次数。
- **CPU（CPU 百分比）**
    - `us` (user): 用户进程消耗的 CPU 时间百分比。
    - `sy` (system): 系统（内核）消耗的 CPU 时间百分比。
    - `id` (idle): CPU 空闲时间百分比。
    - `wa` (wait): CPU 等待 I/O 完成的时间百分比。
    - `st` (steal): 虚拟机偷走的 CPU 时间（仅限虚拟化环境）。

### Shell 基础知识

#### 变量与赋值
- **自定义变量**：使用 `name="value"` 定义，通过 `$name` 引用。
- **环境变量**：脚本中常用的 `$LOGNAME` (当前用户)、`$PATH`、`$PWD` 等。
- **命令替换**：
    - `VAR=$(command)`：这是现代推荐写法，执行命令并将输出赋值给变量。例如 `MEM=$(free -m)`。
    - ``` `command` ```：老式反引号写法。

#### 函数定义
为了让脚本结构清晰，我们将重复调用的逻辑（如系统检测、依赖安装）封装为函数：
```bash
function_name() {
    # 逻辑代码
    return 0
}
```

#### if 条件测试
用于对运行环境进行逻辑判断：
- **字符串比较**：`[ "$A" == "$B" ]`。
- **逻辑非**：`! command` 表示当命令执行失败时触发分支。
- **命令存在性检测**：`command -v <cmd>` 用于检查系统中是否已安装该工具。



### 脚本基础结构实现

#### 操作系统检测函数

为了支持更广泛的发行版并精准识别，我们使用 `/etc/os-release` 文件进行判断。

```bash
# 操作系统检测函数
os_check() {
    if [ -f /etc/os-release ]; then
        . /etc/os-release
        case $ID in
            ubuntu|debian)
                P_M="apt-get"
                ;;
            centos|rhel)
                P_M="yum"
                ;;
            rocky)
                P_M="dnf"
                ;;
            *)
                echo "[报错] 不支持的操作系统: $ID"
                exit 1
                ;;
        esac
    else
        echo "[报错] 无法识别操作系统环境。"
        exit 1
    fi
}
```

在上述优化后的 `os_check` 函数中：
- **`/etc/os-release`**：是现代 Linux 发行版中用于标识系统的标准文件。
- **`.` (source)**：直接引入该文件中的变量（如 `$ID`）。
- **精准适配**：针对 Ubuntu/Debian 使用 `apt-get`，CentOS/RHEL 使用 `yum`，Rocky Linux 使用 `dnf`。

### 标准化与健康检查

#### 标准化日志输出
通过 ANSI 颜色代码区分日志级别，并自动附加时间戳，方便后续排错。

```bash
# 日志格式化函数
log_info() {
    echo -e "\033[32m[INFO] $(date '+%Y-%m-%d %H:%M:%S') - $1\033[0m"
}

log_error() {
    echo -e "\033[31m[ERROR] $(date '+%Y-%m-%d %H:%M:%S') - $1\033[0m"
}
```

#### 磁盘空间健康检查
在脚本执行前检查磁盘空间，防止日志无法写入或系统故障。

```bash
check_disk() {
    # 提取根分区当前利用率
    DISK_USAGE=$(df / | awk 'NR==2 {print $5}' | cut -d'%' -f1)
    if [ "$DISK_USAGE" -gt 90 ]; then
        log_error "根分区磁盘空间不足！当前使用率：${DISK_USAGE}%"
    else
        log_info "磁盘空间正常，当前使用率：${DISK_USAGE}%"
    fi
}
```

#### 网络连通性预检
确保在安装依赖或推送监控结果前，服务器能够访问外部网络。

```bash
check_network() {
    # 检测外网连通性 (以百度为例)
    if ping -c 1 -W 2 www.baidu.com &>/dev/null; then
        log_info "网络连通性正常。"
    else
        log_error "网络连接异常，请检查网关配置。"
    fi
}
```

### 完整的初始化流程 (Main 函数化)

将所有的检测逻辑封装进一个 `init` 或 `main` 流程中，这是编写大型 Shell 脚本的最佳实践。

```bash
#!/bin/bash

# ... 前面定义的各函数 ...

# 初始化环境
init_env() {
    # 1. 权限校验
    if [ "$LOGNAME" != "root" ]; then
        log_error "请使用 root 账号执行脚本。"
        exit 1
    fi

    # 2. 系统版本与包管理器识别
    os_check

    # 3. 基础资源检查
    check_disk
    check_network

    # 4. 依赖安装
    if ! command -v vmstat &>/dev/null; then
        log_info "安装监控组件 (procps)..."
        $P_M install procps -y
    fi
    
    log_info "环境预检完成，开始运行监控脚本。"
}

# 执行入口
init_env
```
