    ```yaml

          - PASSWORD_ACCESS=true

          - USER_NAME=demo
    ```

### 脚本1：推送公钥到各个主机

ssh密钥登录原理：

我们一般使用 PuTTY 等 SSH 客户端来远程管理 Linux 服务器。但是，一般的密码方式登录，容易有密码被暴力破解的问题。所以，一般我们会将 SSH 的端口设置为默认的 22 以外的端口，或者禁用 root 账户登录。其实，有一个更好的办法来保证安全，而且让你可以放心地用 root 账户从远程登录——那就是通过密钥方式登录。

密钥形式登录的原理是：利用密钥生成器制作一对密钥——一只公钥和一只私钥。将公钥添加到服务器的某个账户上，然后在客户端利用私钥即可完成认证并登录。


1. 安装`sshpass`工具

    ```bash
    [root@localhost ~]# yum install -y sshpass
    ```

2. 批量推送公钥脚本

    首先在当前目录下创建 `ports.txt` 文件，内容为需要推送的端口号：
    ```text
    2221
    2222
    ```

    编写脚本 `push_sshkey.sh`：

    ```bash
    #!/bin/bash
    
    # 1. 声明一个空数组
    PORTS=()
    
    # 2. 使用 mapfile 将端口文件每行读入数组
    # -t 参数的作用是去掉每行末尾的“换行符”(\n)
    mapfile -t PORTS < ports.txt
    password="password"

    # 3. 清理 known_hosts 文件，避免重置容器环境后的指纹不一致问题
    > ~/.ssh/known_hosts
    
    # 生成密钥对，若已存在则直接覆盖 (-N "" 表示空密码)
    ssh-keygen -t rsa -N "" -f ~/.ssh/id_rsa <<< "y" >/dev/null
    
    # 4. 遍历数组推送公钥
    for port in "${PORTS[@]}"; do
        
        echo "正在推送公钥至: localhost:$port"
        # 使用 sshpass 自动填充密码 (password 为之前容器设置的密码)
        # -o StrictHostKeyChecking=no 用于跳过首次连接的 yes/no 确认
        sshpass -p "$password" ssh-copy-id -p "$port" -o StrictHostKeyChecking=no "demo@localhost" &>/dev/null
            
        if [ $? -eq 0 ]; then
            echo " -> [localhost:$port] 推送成功"
        else
            echo " -> [localhost:$port] 推送失败"
        fi
    done
    ```

3. 执行推送脚本

    ```bash
    [root@localhost ~]# bash push_sshkey.sh 
    正在推送公钥至: localhost:2221
    -> [localhost:2221] 推送成功
    正在推送公钥至: localhost:2222
    -> [localhost:2222] 推送成功
    ```

4. 再次通过`ssh -p 2221 demo@localhost`远程容器环境，验证是否能免密登录

    ```bash

    b59608ec976e:~$ 
    ```

### 脚本2：多机部署MySQL

使用 `mapfile` 读取端口并利用 SSH 免密环境配合 `sudo -S` 完成自动化安装。

编写脚本 `deploy_mysql.sh`：

```bash
#!/bin/bash

# 1. 声明并读取端口数组
mapfile -t PORTS < ports.txt
password="password"

echo "=== 开始批量安装 MySQL ==="

for port in "${PORTS[@]}"; do
    
    echo "正在 [localhost:$port] 推送指令..."
    # 由于已推送公钥，SSH 登录已免密
    # sudo -S 用于从标准输入接收密码
    ssh -p "$port" -o StrictHostKeyChecking=no "demo@localhost" \
        "echo \"$password\" | sudo -S apk add mysql mysql-client"
        
    if [ $? -eq 0 ]; then
        echo " -> [localhost:$port] 安装成功"
    else
        echo " -> [localhost:$port] 安装失败"
    fi
done
```

### 脚本3：MySQL 初始化与配置管理

安装完成后，需要对 MySQL 进行初始化配置并管理其服务状态。

1. **准备配置文件 `my.cnf`**

    在宿主机创建 `my.cnf` 模板：

    ```ini
    [mysqld]
    user = mysql
    datadir = /var/lib/mysql
    port = 3306
    bind-address = 0.0.0.0
    character-set-server = utf8mb4
    collation-server = utf8mb4_unicode_ci
    ```

2. **批量分发配置并初始化脚本 `init_mysql.sh`**

    ```bash
    #!/bin/bash
    mapfile -t PORTS < ports.txt
    password="password"

    for port in "${PORTS[@]}"
    do
        echo "正在配置 [localhost:$port]..."
        # 1. 推送配置文件 (免密)
        scp -P "$port" my.cnf demo@localhost:/tmp/my.cnf
        
        # 2. 执行初始化命令 (利用 sudo -S 实现自动化)
        ssh -p "$port" "demo@localhost" << EOF
            echo '$password' | sudo -S mkdir -p /etc/mysql
            echo '$password' | sudo -S mv /tmp/my.cnf /etc/mysql/my.cnf
            echo '$password' | sudo -S mysql_install_db --user=mysql --datadir=/var/lib/mysql
            echo '$password' | sudo -S /usr/bin/mariadbd-safe --datadir='/var/lib/mysql'
    EOF
            
        if [ $? -eq 0 ]; then
            echo " -> [localhost:$port] 配置并启动成功"
        fi
    done
    ```

3. **验证 MySQL 运行状态**

    使用 `mysqladmin` 命令验证 `host1` (端口 2221) 容器内的 MySQL 是否运行正常：

    ```bash
    [root@localhost ~]# ssh -p 2221 demo@localhost "echo 'password' | sudo -S mysqladmin -u root ping"

    mysqld is alive
    ```



### 脚本4：配置主从复制 `setup_replication.sh`

我们将 `host1` (2221) 设为 Master，`host2` (2222) 设为 Slave。脚本会自动修改配置、重启服务并关联主从。

```bash
#!/bin/bash

password="password"

echo "=== 1. 配置 Master (2221) ==="
ssh -p 2221 demo@localhost << EOF
    echo '$password' | sudo -S sh -c "echo 'log-bin=mysql-bin' >> /etc/mysql/my.cnf"
    echo '$password' | sudo -S sh -c "echo 'server-id=1' >> /etc/mysql/my.cnf"
    echo '$password' | sudo -S pkill mariadb
    echo '$password' | sudo -S /usr/bin/mariadbd-safe --datadir='/var/lib/mysql' &>/dev/null &
    sleep 3
    mysql -e "CREATE USER 'repl'@'%' IDENTIFIED BY 'repl_password'; GRANT REPLICATION SLAVE ON *.* TO 'repl'@'%';"
EOF

# 获取 Master 的容器 IP 和当前的日志点
master_ip=\$(ssh -p 2221 demo@localhost "hostname -i")
master_log=\$(ssh -p 2221 demo@localhost "mysql -e 'show master status'" | awk 'NR==2{print \$1}')
master_pos=\$(ssh -p 2221 demo@localhost "mysql -e 'show master status'" | awk 'NR==2{print \$2}')

echo "Master 指向: \$master_ip, 日志文件: \$master_log, 位置: \$master_pos"

echo "=== 2. 配置 Slave (2222) ==="
ssh -p 2222 demo@localhost << EOF
    echo '$password' | sudo -S sh -c "echo 'server-id=2' >> /etc/mysql/my.cnf"
    echo '$password' | sudo -S pkill mariadb
    echo '$password' | sudo -S /usr/bin/mariadbd-safe --datadir='/var/lib/mysql' &>/dev/null &
    sleep 3
    mysql -e "CHANGE MASTER TO MASTER_HOST='\$master_ip', MASTER_USER='repl', MASTER_PASSWORD='repl_password', MASTER_LOG_FILE='\$master_log', MASTER_LOG_POS=\$master_pos; START SLAVE;"
EOF

echo "=== 3. 检查同步状态 ==="
ssh -p 2222 demo@localhost "mysql -e 'show slave status\G'" | grep "Running"
```

1. **核心逻辑说明**
    - **`echo >>`**: 使用最简单的追加方式修改配置文件。
    - **`awk 'NR==2'`**: 提取 `show master status` 输出的第二行数据。
    - **变量动态替换**: 脚本自动抓取 Master 状态并填入 Slave 的 `CHANGE MASTER` 命令。

2. **检查同步状态脚本 `check_replication.sh`**

    ```bash
    #!/bin/bash
    echo "--- Slave Status on host2 (2222) ---"
    ssh -p 2222 demo@localhost "mysql -e 'show slave status\G'" 2>/dev/null | grep -E "Slave_IO_Running|Slave_SQL_Running|Seconds_Behind_Master|Last_IO_Error"
    ```
