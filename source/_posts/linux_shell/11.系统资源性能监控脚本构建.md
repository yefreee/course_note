## 第一课：监控脚本框架与基础运行环境检测

**本节课目标：** 搭建脚本基础框架，复习变量、函数及 `if` 判断，并集成 `vmstat` 工具。

### 1. 工具介绍：vmstat

`vmstat` 是监测 CPU、内存、I/O 等系统性能的基础工具。

- **常用命令**：`vmstat 5 1` (每 5 秒输出一次，仅 1 次)。

#### vmstat 输出字段详解

执行 `vmstat` 命令后，输出结果各列含义如下：

- **Procs（进程）**
    - `r` (runnable): 等待运行的进程数。如果长期大于 CPU 核心数，说明 CPU 资源饱和。
    - `b` (blocked): 处在不可中断睡眠状态的进程数，通常是在等待 I/O。
- **Memory（内存）**
    - `swpd`: 使用的虚拟内存（Swap）总量（KB）。如果该值不断变化，说明内存不足。
    - `free`: 空闲的物理内存总量（KB）。
    - `buff`: 用作缓冲区的内存总量（KB），用于合并磁盘写入。
    - `cache`: 用作缓存的内存总量（KB），用于加速文件读取。
- **Swap（交换分区速度）**
    - `si` (swap-in): 每秒从磁盘交换到内存的数据量（KB/s）。
    - `so` (swap-out): 每秒从内存交换到磁盘的数据量（KB/s）。
- **IO（磁盘读写速度）**
    - `bi` (block-in): 每秒从块设备接收到的块数（读操作）。
    - `bo` (block-out): 每秒发送到块设备的块数（写操作）。
- **System（系统）**
    - `in` (interrupt): 每秒中断次数，包括时钟中断。
    - `cs` (context switch): 每秒上下文切换次数。
- **CPU（CPU 百分比）**
    - `us` (user): 用户进程消耗的 CPU 时间百分比。
    - `sy` (system): 系统（内核）消耗的 CPU 时间百分比。
    - `id` (idle): CPU 空闲时间百分比。
    - `wa` (wait): CPU 等待 I/O 完成的时间百分比。
    - `st` (steal): 虚拟机偷走的 CPU 时间（仅限虚拟化环境）。

### 2. Shell 基础知识

#### 2.1 变量与赋值
- **自定义变量**：使用 `name="value"` 定义，通过 `$name` 引用。
- **环境变量**：脚本中常用的 `$LOGNAME` (当前用户)、`$PATH`、`$PWD` 等。
- **命令替换**：
    - `VAR=$(command)`：这是现代推荐写法，执行命令并将输出赋值给变量。例如 `MEM=$(free -m)`。
    - ``` `command` ```：老式反引号写法。

#### 2.2 函数定义
为了让脚本结构清晰，我们将重复调用的逻辑（如系统检测、依赖安装）封装为函数：
```bash
function_name() {
    # 逻辑代码
    return 0
}
```

#### 2.3 if 条件测试
用于对运行环境进行逻辑判断：
- **字符串比较**：`[ "$A" == "$B" ]`。
- **逻辑非**：`! command` 表示当命令执行失败时触发分支。
- **命令存在性检测**：`command -v <cmd>` 用于检查系统中是否已安装该工具。



### 3. 脚本基础结构实现

#### 操作系统检测函数

```bash
#!/bin/bash

# 操作系统检测函数
os_check() {
    if [ -e /etc/redhat-release ]; then
        REDHAT=$(cat /etc/redhat-release |cut -d' ' -f1)
        P_M=yum
    else
        DEBIAN=$(cat /etc/issue |cut -d' ' -f1)
        P_M=apt-get
    fi
}
```

在上述 `os_check` 函数中使用了 `cut` 命令来动态提取系统名称。
- **常用参数**：
    - `-d`：指定分隔符（Delimiter）。
    - `-f`：指定提取第几个字段（Field）。
- **实战案例**：`cut -d' ' -f1` 表示以空格为分隔符，截取第一个区域的内容。
- **通用示例**：
    ```bash
    # 提取 /etc/passwd 中的用户名（以冒号分隔的第一列）
    head -n 5 /etc/passwd | cut -d: -f1
    ```

#### 权限检查与依赖检测

```bash
# 1. 权限检查
if [ "$LOGNAME" != root ]; then
    echo "[报错] 请使用 root 账号执行。"
    exit 1
else
    echo "[确认] 当前已使用 root 账号执行。"
fi

# 2. 依赖检测 (集成 vmstat)
if ! command -v vmstat &>/dev/null; then
    echo "[提示] 未找到 vmstat 命令，准备安装..."
    os_check
    $P_M install procps -y
else
    echo "[确认] 系统已存在 vmstat 命令，无需安装。"
fi
```

## 第二课：交互式菜单与存储资源监控

**本节课目标：** 使用循环和选择语句构建交互菜单，集成 `iostat` 及磁盘空间监控分析。

### 1. 核心工具：iostat & 磁盘命令
- **iostat**：监测磁盘 I/O 负载。
    - **常用参数**：`-x` (显示扩展统计信息)，`-k` (以 KB 为单位显示)。
    - **核心关注指标**：
        - `%util`：**磁盘忙碌百分比**。若该值接近 100%，表示磁盘接近满负荷。
        - `await`：**平均等待时间**（ms）。反映磁盘响应速度，值越大通常代表延迟越高。
        - `rkB/s` / `wkB/s`：每秒读取/写入的数据量。
    - **详细指标说明**:
        - `rrqm/s` / `wrqm/s`：每秒合并到设备队列的读/写请求数。
        - `r/s` / `w/s`：每秒完成的读/写次数（即 **IOPS**）。
        - `rMB/s` / `wMB/s`：每秒读/写的吞吐量（单位为 MB）。
        - `avgrq-sz`：平均每个 I/O 请求的大小（扇区数）。
        - `avgqu-sz`：平均 I/O 队列长度。
        - `r_await` / `w_await`：平均每个读/写操作的耗时（ms）。
- **df & fdisk**：查看文件系统硬盘利用率及真实磁盘总量。

### 2. Shell 核心语法
- **select in 循环**：用于生成交互式的编号菜单。
    - **基本语法**：
        ```bash
        select variable in list
        do
            commands
        done
        ```
    - **关键特性**：
        1. 它会自动显示菜单项并带上序号。
        2. 它会无限循环，直到在循环体中遇到 `break` 语句或按下 `Ctrl+D`。
        3. 通常与 `case in` 结合使用来处理用户选择。
    - **示例代码**：
        ```bash
        #!/bin/bash
        echo "请选择您的操作系统："
        select choice in "Linux" "Windows" "Mac OS" "Quit"
        do
            case $choice in
                "Linux")
                    echo "您选择了开源系统！"
                    ;;
                "Quit")
                    break
                    ;;
                *)
                    echo "无效选择"
                    ;;
            esac
        done
        ```
- **case in 语句**：根据用户输入执行不同操作。
- **while 循环**：用于让菜单持续运行或多次采样。
- **echo -e**：给输出信息添加颜色标记 (如 `\033[32m` 绿色)。

### 3. 课程实践：菜单构建与存储模块

#### 菜单构建


```bash
# 模拟主循环
while true; do
    select input in cpu_load disk_load disk_use quit; do
        case $input in
            cpu_load)
                # 利用 vmstat 解析 CPU
                UTIL=$(vmstat | awk '{if(NR==3)print 100-$15"%"}')
                echo "[结果] 总利用率: $UTIL"
                break ;;
            disk_load)
                # 使用 iostat 监控 I/O
                iostat -x -k | awk '/^[v|s]/{OFS=": ";print $1,$NF"%"}'
                break ;;
            quit) exit 0 ;;
        esac
    done
done
```

## 第三课：进阶数据分析：网络、进程与 AWK 高级应用

**本节课目标：** 学习多列文本处理和数组统计，实现 TCP 状态分析、Top 10 进程监控及流量统计。

### 1. 核心工具：netstat & ps & sort
- **netstat**：查看网络连接状态和协议统计。
- **ps aux**：获取当前所有进程状态快照。
- **sort**：对结果进行排序（如 `sort -nr` 按数值倒序）。

### 2. AWK 高级扩展
- **AWK 数组**：用于分类统计。
    - `shells[$NF]++`：对最后一列统计出现次数。
- **AWK for 循环**：遍历统计后的数组结果。

### 3. 课程实践：高级模块实现

#### TCP 状态统计
```bash
netstat -antp | awk '{status[$6]++} END{for (i in status){print i, status[i]}}'
```

#### 资源占用前 10 进程统计
```bash
ps aux | awk '{if($3 > 0.1) printf "PID: %s CPU: %s%% \n", $2, $3}' | sort -k4 -nr | head -10
```

#### 实时流量监控
通过 `ifconfig` 在 1 秒间隔内读取 Bytes 差值并换算带宽。
